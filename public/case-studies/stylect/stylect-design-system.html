03 — Foundations

Motion Spec

Fashion doesn't snap. Every transition earns its time. The faster something moves, the less authority it carries. Reserve the slowest reveals for the highest-stakes moments — stylist card opening, shortlist arrival.

Interaction Duration Easing PRD Screen Notes
Page / screen transition 550ms cubic(0.4,0,0.2,1) All screens Opacity fade + 24px Y rise. No horizontal slides.
Onboarding dark→light 800ms cubic(0.4,0,0.2,1) 1.3 → 2.1 Background washes from #080808 to #F5F1EB over account creation confirm.
Card hover lift 200ms cubic(0.4,0,0.2,1) Stylist card, Shortlist translateY(–2px) + shadow emergence.
Shortlist arrival (3 cards) 550ms stagger 120ms cubic(0.34,1.56,0.64,1) Screen 2.4 Spring ease — slight overshoot. Cards feel placed by hand, not loaded.
Brief card open 350ms cubic(0.4,0,0.2,1) Screen 2.2 Height expands from 0 with content fade.
Rule / line draw 1200ms cubic(0.4,0,0.2,1) Load states scaleX from 0 to 1 with transform-origin: left. Onboarding only.
Stylist card "session open" 600ms cubic(0.4,0,0.2,1) Screen 2.3 → 2.4 Card expands to fill viewport — curtain rising, not page push.
Keep/Return swipe 350ms cubic(0.4,0,0.2,1) Screen 2.5 / F4 loop Card exits with translateX(±80px) + opacity 0. Never rotate — garments don't toss.
Toast enter 350ms cubic(0.34,1.56,0.64,1) All screens Slides in from right 24px. Auto-dismiss 4s.
Progress bar fill 550ms cubic(0.4,0,0.2,1) Brief Builder 1.2 Width transition. Label fades in at completion.
Button hover 200ms cubic(0.4,0,0.2,1) All CTAs translateY(–1px). Never scale — scaling looks cheap.
SLA timer pulse 1000ms infinite ease-in-out Stylist dashboard Subtle opacity 1 → 0.6 on the timer text when < 30 min remain.

20 — Reference

Usage Rules

The system earns authority through restraint. Every rule exists to protect the stylist's voice and the garment's presence. These are decisions, not suggestions.

Typography Rules
+ Do
  • Use Bodoni italic for stylist notes and occasion names
  • Use Space Mono for all prices, dates, SKUs, status
  • One D1 or D2 heading per screen maximum
  • Let negative space carry hierarchy weight
  • Italic accent color for emotional beats
× Don't
  • Mix Bodoni with any other serif
  • Use bold weight in body copy
  • Center-align body text on any screen
  • Use Bodoni for UI labels or nav items
  • Stack two D-scale headings adjacently
Color Rules
+ Do
  • Use dark mode only for screens 1.1–1.3
  • Accent (#C9B99A) for one element per section
  • Sage for success and availability states
  • Error red (#C0412A) for destructive actions only
  • Transition from dark→light on account confirm
× Don't
  • Use dark mode after onboarding completes
  • Use accent on two CTAs in the same view
  • Use pure white (#FFFFFF) as a background
  • Add color to convey information without a text label
  • Introduce new brand colors without system approval
Component Rules
+ Do
  • Use pill radius (999px) for all buttons and chips
  • Use r-lg (16px) for cards, r-xl (20px) for brief/hero cards
  • Use glass panels over full-bleed photos — blur(20px) minimum
  • One primary CTA per screen at all times
  • Show SLA timer on every active brief card
  • Stylist avatar always visible in session views
× Don't
  • Use glass on solid flat backgrounds — it needs depth behind it
  • Mix radius sizes on sibling cards in the same view
  • Add cart badge during any active chat session
  • Use rounded corners on the wordmark or rule elements
  • Add star-only ratings without session count context
  • Interrupt active chat with modal overlays
