01 — Problem

On makes the world's best running hardware.
It has no software home.

Strava owns the activity layer. Nike Run Club owns the social layer. On owns the product — but has no owned digital surface where community, performance identity, and brand meaning converge.

→
The Core Gap On's digital presence is transactional (buy) or informational (learn). There is no surface that captures the On runner's identity, connects them to others, or gives meaning to their miles in an On-specific context. Every competitive insight is gifted to third-party platforms.
Evidence of the Gap
Signal Implication Priority
On has no owned activity-tracking app Zero behavioral data on how runners actually use On product Critical
Strava integration exists but is external On's most engaged runners live in a competitor's interface Critical
On's community is diffuse across Reddit, Strava, Instagram No owned conversation surface; no loop-back to brand High
Club / group running growing globally post-COVID Massive category moment; no On-specific community infrastructure High
CloudMonster, Cloudflow, Cloudstratus have distinct user profiles Product-led community segmentation is possible but untapped Medium

03 — Goals & Success

What winning looks like.

Goals are anchored to three horizons: activation, engagement, and brand value creation.

Product Goals
Own the post-purchase identity moment When a runner laces up their new On shoes, Apex should be the first digital thing they reach for — not Strava.
Capture behavioral data On doesn't have Activity cadence, shoe rotation, preferred distances, social graph — create a behavioral dataset that informs product, marketing, and loyalty decisions.
Build community with On aesthetics, not generic fitness UX Every interaction should feel as Swiss-engineered as the product itself. No gamification for its own sake. No badges that feel like participation trophies.
Create a commerce loop without being a shopping app Contextual gear discovery, shoe mileage tracking, and Cyclon program integration — commerce as utility, not interruption.
Support On's move into experiential retail and events Club runs, pop-up events, and On Apex community challenges become discoverable and RSVP-able within the platform.

Design System — Foundation

Molecular Design System
On Apex Edition

Swiss engineering applied to mobile UI. Zero border-radius. 4px grid. DM Sans as a Swiss grotesque stand-in. Restraint is the brand. Every component earns its presence.

◆
Atom The indivisible unit. Color token, spacing value, type style, motion curve. Cannot be broken down further. No opinions about context.
◆◆
Molecule Two or more atoms forming a functional unit. A button (atom: ink + surface + type + border). A metric card (atoms: type + spacing + border). Has a single purpose.
◆◆◆
Organism A complex UI region. The activity feed. The shoe garage card. The nav bar. Composed of molecules. Has context-specific behavior.
→
Design Principle If a component can't justify its presence with a functional argument, it doesn't ship. Beauty is a byproduct of precision, not decoration.

Why Dual Mode

Running happens across the full luminosity spectrum — pre-dawn runs, midday sun, evening cooldowns. Dark mode is not a preference accommodation. It's an environmental adaptation. The On Apex color system treats both modes as first-class citizens, not a light-mode app with a dark theme bolted on.

○
Light — Swiss Precision Clean white surfaces. Maximum product photography contrast. Readable in outdoor daylight. The default for discovery, onboarding, and browsing states.
●
Dark — Run Mode Near-black surfaces with volt accents. Reduces screen glare during active running. Volt reads even more powerfully here. The native state of running culture.

Design System — Behavior

Interaction Patterns

Advanced mobile UX patterns specified for On Apex. Each pattern has a gesture, a duration, and a purpose.

Haptic Tap
All primary CTAs trigger light haptic on tap. Confirm actions trigger medium. Destructive actions trigger warning pattern. 0ms delay — instant.
Swipe Dismiss
Notification sheets and contextual menus dismiss on horizontal swipe ≥80px. Velocity threshold: 300px/s. Spring rebound if threshold not met.
Pull to Refresh
Feed and activity log pull-to-refresh. Threshold: 72px. Loading state: single line 2px scanning animation. No spinner — precision over skeuomorphism.
Long Press
Activity cards support long press (400ms) to reveal: share, edit, delete. Blur background, surface modal. Contextual — no global long press behavior.
Scroll Snap
Horizontal carousels (shoe garage, suggested routes) use CSS scroll-snap-type: mandatory. Each card snaps to alignment. No momentum scrolling past endpoint.
Sheet Modal
Bottom sheets animate up from 0 to final position in 380ms ease-decelerate. Max 90% screen height. Drag handle visible. Scrim dims to 40% opacity.
Toast Notification
Run saved, shoe registered, club joined. 3s duration. Slide up from bottom, 16px above nav bar. Auto-dismiss. No action required — acknowledgment only.
Skeleton Loading
Content areas show skeleton before data loads. Animated shimmer scan — left to right, 1.5s cycle. Matches layout of real content exactly. No spinners.
Map Expand
Route preview maps expand on tap to full-screen. Transition: scale from thumbnail origin point, 380ms spring. Back button returns to origin at same speed.
