EM DASH REMOVAL LOG
===================

What changed
------------
Across the portfolio (public/case-studies HTML, src/app pages and components, globals.css comments,
TOMORROW.txt, carolyn-image-inventory .txt, and src/data/mirrorball-songs.json), unicode em dashes (—) and
HTML entities (&mdash;) in user-facing copy were replaced with commas, semicolons, colons, parentheses, or
middle dots (·) so sentences stay grammatical.

Automation
----------
Most passes used scripts/remove_emdashes.py (fixed iter_files indentation bug 2025-03-30). That script is
unsafe for arbitrary JSON (paired-dash heuristics mangled strings), so .json is not bulk-processed.

Not in scope
------------
- AUDIT-GAMEPLAN.md (internal checklist; still contains em dashes)
- Generated out/ folder
- Root package-lock.json / tsconfig (no em dashes)
